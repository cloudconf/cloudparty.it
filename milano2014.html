Un sistema di Continuous Integration fra progetti eterogenei

di Simone Pasquini (OneBip, )

Continuos Integration si riferisce alla pratica di aggiungere nuove feature nel codice e di testarlo non solo in modo isolato ma anche in piena integrazione con gli altri progetti durante il ciclo di sviluppo del software.
Implementare un sistema automatizzato di CI prevede un repository con il codice sorgente utilizzato durante la fase di sviluppo ed uno strumento di CI che continui a fare polling del codice e controllarlo tramite i test in integrazione. Ma non basta quando ci sono progetti eterogenei che collaborano fra di loro, in questo talk vedremo specificatamente questo aspetto.

15:10 - 15:45

Strategie di deploy con Fabric e Saltstack

di Luca Cipriani (SysApp Architect, )

Come deployare in parallelo su un numero elevato di server. Differenza fra push e pull deploy, atomic deploy, uso di git e deploy di binari. Differenze nel deploy fra Google Compute Engine e Amazon AWS. Gli strumenti usati sono Fabric e Saltstack.

15:45 - 16:00

Break!

16:00 - 16:50

Scalare l'infrastruttura di un'applicazione web limitando le modifiche al codice

di Walter Dal Mut (Corley srl, )

In questo mini-workshop vedremo come fare il deploy di un'applicazione PHP-MySQL verso un'infrastruttura scalabile sfruttando il cloud ECS basato su OpenStack senza dover modificare il codice sorgente dell'applicazione. Vedremo come astrarre la scalabilità delle letture dei server database MySQL, distribuire le sessioni utenti e storage di cache, aggiungere load balancer per tenere sotto contorollo l'alta disponibilità della nostra struttura.

16:50 - 17:30

Vagrant, riutilizzo dell'infrastruttura

di Gianluca Arbezzano, (Corley srl, )

Poter scriptare gli ambienti per configurare la propria infrastruttura è una pratica essenziale in cloud e non solo. Se in più potessimo riutilizzare le stesse configurazioni per lo sviluppo locale il valore aggiunto sarebbe notevole. Proveremo quindi a riutilizzare gli stessi script usati nel talk precedente durante il deploy di un applicazione in cloud per configurare il nostro ambiente di sviluppo, scoprendone problematiche e benefici.

17:30
